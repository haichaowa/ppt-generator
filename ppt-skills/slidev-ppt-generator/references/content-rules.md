# PPT 内容组织规则

本文档定义如何将用户输入的文字内容组织为优秀的 Slidev 演示文稿。

---

## 一、核心原则

### 1. 一页一观点
- 每张幻灯片只传达一个核心观点
- 避免信息过载
- 观众一次只能记住 3-5 个要点

### 2. 金字塔原理
- **结论先行**：先说核心观点，再说支撑论据
- **以上统下**：上层观点统领下层论据
- **归类分组**：相关内容放在一起

### 3. 少即是多
- **文字精简**：每页不超过 50 字（中文）
- **要点控制**：3-5 条为佳
- **留白呼吸**：适当留白让内容透气

---

## 二、页面数量建议

| 演讲时长 | 推荐页数 | 说明 |
|---------|---------|------|
| 3-5 分钟 | 6-10 页 | 短分享，内容精简 |
| 5-10 分钟 | 10-15 页 | 中等长度，适当展开 |
| 10-20 分钟 | 15-25 页 | 深入讲解 |
| 20-40 分钟 | 25-45 页 | 完整演讲 |

---

## 三、标准结构模板

```
1. 封面页（1页）
2. 目录页（1页，可选）
3. 章节 1（1页章节分隔 + N页内容）
4. 章节 2（1页章节分隔 + N页内容）
5. 章节 3（1页章节分隔 + N页内容）
6. ...
7. 结束页（1页）
```

---

## 四、内容特征 → Slidev 布局映射

| 原始内容特征 | 推荐布局 | Frontmatter 配置 |
|-------------|---------|-----------------|
| 演讲标题 + 副标题 | `center` | `layout: center` |
| 章节分隔/编号切换 | `section` | `layout: section` |
| 观点 + 3-5 论据 | `default` | （默认布局） |
| 对比/优缺点分析 | `two-cols` | `layout: two-cols` + `::right::` |
| 带标题的对比 | `two-cols-header` | `layout: two-cols-header` + `::left::`+`::right::` |
| 图片+文字说明 | `image-left/right` | `layout: image-left` + `image: 路径` |
| 全屏图片 | `image` | `layout: image` + `image: 路径` |
| 代码演示 | `default` | 代码块 + 行高亮 |
| 名言金句 | `quote` | `layout: quote` |
| 关键数据展示 | `fact` | `layout: fact` |
| 核心结论陈述 | `center` 或 `statement` | `layout: statement` |
| 演讲者介绍 | `intro` | `layout: intro` |
| 结束致谢 | `end` | `layout: end` |

## 五、Slidev 内容增强技巧

### 渐进展示（v-click）
用 `v-click` 指令逐步揭示内容，避免一次性呈现太多信息：
```html
<div v-click>第一个要点</div>
<div v-click>第二个要点</div>
<div v-click>第三个要点</div>
```

或使用 `<v-clicks>` 组件自动为列表项添加动画：
```html
<v-clicks>
- 要点一
- 要点二
- 要点三
</v-clicks>
```

### 双栏布局
使用 `::right::` 插槽创建对比页：
```markdown
---
layout: two-cols
---
# 左侧标题
左侧内容

::right::
# 右侧标题
右侧内容
```

### 代码高亮
使用行号高亮和点击式高亮：
````markdown
```ts {1,3-5}
// 第1行和第3-5行高亮
const name = 'slidev'
const version = '1.0'
const description = 'slides for developers'
```
````

点击式逐步高亮：
````markdown
```ts {1|2-3|all}
// 点击1: 只高亮第1行
// 点击2: 高亮第2-3行
// 点击3: 高亮所有行
```
````

### 演讲者备注
在幻灯片末尾使用 HTML 注释添加备注：
```markdown
# 标题

内容

<!-- 这段是演讲者备注，只在演示者视图中显示 -->
```

### 过渡动画
在 frontmatter 中指定幻灯片过渡效果：
```yaml
---
transition: fade-out
---
```

可选值：`fade`、`fade-out`、`slide-left`、`slide-right`、`slide-up`、`slide-down`、`zoom`

## 六、内容密度建议

| 演讲类型 | 内容密度 | 说明 |
|---------|---------|------|
| 教程类 | 中等 | 步骤清晰，留白适当 |
| 分享类 | 较高 | 信息量大，节奏快 |
| 产品类 | 较低 | 视觉为主，文字精简 |
| 技术类 | 中高 | 代码+解释结合 |

## 七、检查清单

### 内容检查
- [ ] 每页不超过 50 字
- [ ] 列表项 3-5 条
- [ ] 一页一观点
- [ ] 标题简洁有力

### 格式检查
- [ ] 布局选择合理
- [ ] frontmatter 配置正确
- [ ] `::right::`/`::left::` 插槽位置正确
- [ ] 代码块高亮语法正确
- [ ] 代码超过 15 行时使用了 `maxHeight` 防止溢出

### 结构检查
- [ ] 有封面和结束页
- [ ] 章节分隔清晰
- [ ] 页面数量合适
- [ ] 动画节奏合理
