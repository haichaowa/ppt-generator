# PPT Markdown 格式规范

本文档定义了 PPT 生成器所使用的 Markdown 格式规范。

## 设计原则

1. **不使用 `>` 引用块表示副标题**（避免与 Markdown 引用语法冲突）
2. **使用标准 Markdown 标题层级**：`#` 主标题, `##` 副标题, `###` 子标题
3. **使用 HTML 注释 `<!-- -->` 作为特殊标记**
4. **使用 YAML 风格的元数据行**（如 `作者 | 日期`）

## 通用规则

### 幻灯片分隔

每个幻灯片必须以 HTML 注释标记类型：

```markdown
<!-- .slide: type=xxx -->

# 内容...
```

### 标题层级

| 层级 | 用途 |
|-----|-----|
| `#` | 主标题（封面、章节、结束页） |
| `##` | 副标题或幻灯片标题 |
| `###` | 子标题或卡片标题 |
| `-` | 列表项 |

---

## 11 种幻灯片类型

### 1. 封面页 (cover)

封面页用于视频开头，包含主标题、副标题和元数据。

```markdown
<!-- .slide: type=cover -->

# 视频主标题

## 副标题或Slogan

作者 | 2025.02.18
```

**结构说明**：
- `#` 主标题：大字显示，带霓虹发光效果
- `##` 副标题：中等字号，辅助说明
- 底部元数据：`作者 | 日期` 格式

---

### 2. 目录页 (toc)

目录页展示视频内容大纲。

```markdown
<!-- .slide: type=toc -->

# 本期内容

## 01 背景介绍
## 02 核心概念
## 03 实战演示
## 04 总结回顾
```

**结构说明**：
- `#` 标题：目录标题
- `## 编号 内容`：每个目录项使用 `##` 并带编号

---

### 3. 章节页 (chapter)

章节页用于分隔主要内容段落。

```markdown
<!-- .slide: type=chapter -->

# 01

## 章节名称
```

**结构说明**：
- `#` 数字：章节编号，会放大显示
- `##` 名称：章节描述文字

---

### 4. 内容页 (content)

标准内容页，适用于大多数场景。

```markdown
<!-- .slide: type=content -->

## 标题

描述文字段落。

- 要点一
- 要点二
- 要点三
```

**结构说明**：
- `##` 标题：页面标题
- 段落：描述性文字
- `-` 列表：3-5个要点

---

### 5. 双栏页 (two-column)

用于对比、优缺点分析等场景。

```markdown
<!-- .slide: type=two-column -->

## 对比标题

### 左侧标题

左侧内容描述

- 左侧要点

---

### 右侧标题

右侧内容描述

- 右侧要点
```

**结构说明**：
- `##` 标题：整体标题
- `###` 子标题：左右两栏各自标题
- `---` 分隔符：用于分隔左右两栏内容

**注意**：`---` 必须独立一行，前后有空行。

---

### 6. 代码页 (code)

用于展示代码片段。

```markdown
<!-- .slide: type=code -->

## 代码标题

```python
import asyncio

async def fetch_data():
    await asyncio.sleep(1)
    return "data"
```

### 使用 asyncio 实现异步
```

**结构说明**：
- `##` 标题：代码展示标题
- 围栏代码块：使用三个反引号包裹代码，指定语言
- `###` 说明：代码下方说明文字

**支持的语言**：python, javascript, typescript, java, cpp, go, rust, html, css, sql, bash 等

---

### 7. 数据页 (data)

用于展示数据表格。

```markdown
<!-- .slide: type=data -->

## 数据标题

| 项目 | 数值 | 增长 |
|-----|-----|-----|
| 用户 | 100万 | +25% |
| 收入 | 500万 | +40% |
| 利润 | 200万 | +35% |
```

**结构说明**：
- `##` 标题：数据展示标题
- Markdown 表格：标准表格语法

**表格语法**：
- 第一行：表头
- 第二行：分隔线（`|---|`）
- 后续行：数据行

---

### 8. 时间轴页 (timeline)

用于展示时间节点和发展历程。

```markdown
<!-- .slide: type=timeline -->

## 发展历程

### 2020.01 项目启动
首次发布原型版本

### 2021.06 正式上线
用户突破10万

### 2022.12 规模化
用户突破100万
```

**结构说明**：
- `##` 标题：时间轴标题
- `### 日期 标题`：时间节点，日期和标题用空格分隔
- 段落：节点描述

**日期格式**：支持 `YYYY.MM`, `YYYY-MM`, `YYYY年MM月` 等格式

---

### 9. 网格页 (grid)

用于展示并列卡片，仅支持 2x2（4卡片）或 3x3（9卡片）布局。

#### 2x2 示例

```markdown
<!-- .slide: type=grid -->

## 核心功能

### 功能一
- icon: ◉
- color: cyan
- desc: 功能描述文字

### 功能二
- icon: ◈
- color: purple
- desc: 功能描述文字

### 功能三
- icon: ◆
- color: pink
- desc: 功能描述文字

### 功能四
- icon: ▣
- color: orange
- desc: 功能描述文字
```

#### 3x3 示例

```markdown
<!-- .slide: type=grid -->

## 九大特性

### 特性一
- icon: ◉
- color: cyan
- desc: 描述一

### 特性二
- icon: ◈
- color: purple
- desc: 描述二

...（共9个卡片）
```

**卡片属性**：

| 属性 | 说明 | 示例 |
|-----|-----|-----|
| `icon` | 图标字符 | ◉, ◈, ◆, ▣, ●, ★, ▲, ■, ◆ |
| `color` | 颜色名称 | cyan, purple, pink, orange, green, blue |
| `desc` | 描述文字 | 功能描述 |

**约束**：
- 卡片数量必须是 **4 或 9**
- 每个卡片必须包含 `icon`, `color`, `desc` 三个属性

---

### 10. 金句页 (quote)

用于展示名人名言或金句。

```markdown
<!-- .slide: type=quote -->

## 引用金句

> 设计不仅仅是外观。设计是它如何工作。

### — 史蒂夫·乔布斯
```

**结构说明**：
- `##` 标题：金句页标题
- `>` 引用：真正的 Markdown 引用语法
- `### — 作者`：作者署名，使用 `—` 开头

---

### 11. 结束页 (end)

用于视频结尾，引导关注。

```markdown
<!-- .slide: type=end -->

# 感谢观看

## 点赞关注不迷路

作者 | 联系方式
```

**结构说明**：
- `#` 主标题：感谢语
- `##` 副标题：引导关注语
- 底部元数据：`作者 | 联系方式` 格式

---

## 类型速查表

| 类型 | 标题层级 | 特殊语法 | 用途 |
|-----|---------|---------|------|
| `cover` | `#` `##` | 底部元数据行 | 封面页 |
| `toc` | `#` `##` 编号列表 | - | 目录页 |
| `chapter` | `#` 数字 `##` 描述 | - | 章节分隔 |
| `content` | `##` `###` | 列表 `-` | 标准内容 |
| `two-column` | `##` `###` | `---` 分隔 | 双栏对比 |
| `code` | `##` `###` | 围栏代码块 | 代码演示 |
| `data` | `##` | 表格 `\|` | 数据展示 |
| `timeline` | `##` `### 日期 标题` | - | 时间轴 |
| `grid` | `##` `###` | `- key: value` (仅2x2/3x3) | 网格卡片 |
| `quote` | `##` `###` | `>` 引用 | 金句展示 |
| `end` | `#` `##` | 底部元数据行 | 结束页 |

---

## 最佳实践

### 1. 内容精简

- 每页文字不超过 50 字
- 列表项控制在 3-5 条
- 一页只传达一个核心观点

### 2. 视觉层次

- 使用 `#` 突出最重要的内容
- 使用 `##` 作为页面主要信息
- 使用 `###` 补充说明

### 3. 格式一致

- 相同类型的页面使用相同的格式
- 编号、日期格式保持统一
- 元数据格式保持一致

### 4. 可读性

- 避免过长的段落
- 使用列表代替长句
- 适当使用空行分隔内容
