# PPT 内容转换规则

本文档定义了如何将原始文本内容转换为 PPT Markdown 格式的规则。

---

## 一、优秀 PPT 的核心原则

### 1. 一页一观点

- 每张幻灯片只传达一个核心观点
- 避免信息过载，保持焦点清晰
- 观众一次只能记住 3-5 个要点

**示例**：

❌ **错误**：一页同时讲"什么是异步"和"asyncio 用法"
✅ **正确**：分两页，一页讲概念，一页讲用法

### 2. 金字塔原则

- **结论先行**：先说核心观点，再说支撑论据
- **以上统下**：上层观点统领下层论据
- **归类分组**：相关内容放在一起

**示例**：

❌ **错误**：先讲历史，再讲现状，最后才说结论
✅ **正确**：先说"Python 异步编程能提升性能 10 倍"，再解释原因

### 3. 少即是多

- **文字精简**：每页不超过 50 字
- **要点控制**：3-5 条为佳
- **留白呼吸**：适当留白让内容透气

## 二、内容结构化规则

### 页面数量建议

| 视频时长 | 推荐页数 | 说明 |
|---------|---------|------|
| 3-5 分钟 | 6-10 页 | 短视频，内容精简 |
| 5-10 分钟 | 10-15 页 | 中等长度，适当展开 |
| 10-20 分钟 | 15-25 页 | 长视频，深入讲解 |

### 标准结构模板

```
1. 封面页（1页）- 吸引注意力
2. 目录页（1页，可选）- 展示结构
3. 章节 1（1页章节 + N页内容）
4. 章节 2（1页章节 + N页内容）
5. 章节 3（1页章节 + N页内容）
6. ...
7. 结束页（1页）- 引导关注
```

### 结构规划示例

**10 分钟视频：Python 异步编程入门**

```
01. 封面页
02. 目录页
03. 章节 1：什么是异步
    - 概念解释页
    - 同步 vs 异步对比页
04. 章节 2：asyncio 基础
    - 基本语法页
    - 代码演示页
05. 章节 3：实战应用
    - 应用场景页
    - 最佳实践页
06. 结束页
```

## 三、内容类型匹配规则

根据原始内容的特征，选择合适的幻灯片类型。

### 匹配规则表

| 原始内容特征 | 推荐类型 | 判断依据 |
|-------------|---------|---------|
| 标题 + 一句话介绍 | `cover` | 开头部分，包含标题和副标题 |
| 3-6 个并列项目 | `toc` / `grid` | 多个同级项目，无层级关系 |
| 编号标题（01/02/03） | `chapter` | 明确的章节分隔 |
| 观点 + 3-5 个论据 | `content` | 一个核心观点配多个支撑点 |
| 对比/优缺点 | `two-column` | 明确的二元对比关系 |
| 代码片段 | `code` | 包含代码块 |
| 数据表格 | `data` | 数值型数据，适合表格展示 |
| 时间节点列表 | `timeline` | 按时间顺序排列的事件 |
| 名人名言 | `quote` | 引用语句 |
| 致谢/引导关注 | `end` | 结尾部分 |

### 详细说明

#### 封面页 (cover)

**触发条件**：
- 文档开头
- 包含主标题和副标题
- 需要展示作者信息

**内容要点**：
- 主标题：简短有力，8 字以内
- 副标题：补充说明，一句话
- 元数据：作者、日期

#### 目录页 (toc)

**触发条件**：
- 3-6 个主要部分
- 需要展示整体结构
- 视频超过 5 分钟

**内容要点**：
- 每项使用编号
- 文字简洁，5 字以内
- 按顺序排列

#### 章节页 (chapter)

**触发条件**：
- 明显的段落分隔
- 编号标题（01、02、03）
- 内容主题切换

**内容要点**：
- 数字醒目
- 章节名称简洁
- 一页只显示一个章节

#### 内容页 (content)

**触发条件**：
- 一个核心观点
- 3-5 个支撑论据
- 最常用的类型

**内容要点**：
- 标题是核心观点
- 列表是支撑论据
- 每项 10 字以内

#### 双栏页 (two-column)

**触发条件**：
- 明确的对比关系
- 优缺点分析
- 两种方案对比

**内容要点**：
- 左右平衡
- 对应要点对齐
- 标题明确对比维度

#### 代码页 (code)

**触发条件**：
- 包含代码示例
- 需要语法高亮
- 技术演示

**内容要点**：
- 代码精简，15 行以内
- 高亮关键部分
- 添加简要说明

#### 数据页 (data)

**触发条件**：
- 数值型数据
- 多行多列
- 适合表格展示

**内容要点**：
- 3-5 行数据
- 列数不超过 4
- 数据有对比性

#### 时间轴页 (timeline)

**触发条件**：
- 时间顺序事件
- 发展历程
- 项目里程碑

**内容要点**：
- 3-5 个节点
- 日期格式统一
- 每个节点有描述

#### 网格页 (grid)

**触发条件**：
- 4 或 9 个并列项目
- 每个项目有图标、颜色、描述
- 功能特性展示

**内容要点**：
- 只支持 2x2 或 3x3
- 每个卡片结构一致
- 图标和颜色搭配

#### 金句页 (quote)

**触发条件**：
- 名人名言
- 核心理念
- 值得传播的观点

**内容要点**：
- 引用有出处
- 文字精炼
- 适当使用增加传播性

#### 结束页 (end)

**触发条件**：
- 文档结尾
- 需要引导关注
- B站视频必备

**内容要点**：
- 感谢语简洁
- 引导关注明确
- 联系方式清晰

## 四、B站视频 PPT 特殊规则

### 节奏控制

| 页面类型 | 建议停留时间 | 说明 |
|---------|-------------|------|
| 封面页 | 2-3 秒 | 快速展示，吸引注意 |
| 目录页 | 3-5 秒 | 快速浏览结构 |
| 章节页 | 1-2 秒 | 短暂停顿，切换话题 |
| 内容页 | 30-60 秒 | 核心内容，充分讲解 |
| 代码页 | 60-90 秒 | 代码需要更多时间理解 |
| 金句页 | 5-10 秒 | 短暂强调 |
| 结束页 | 3-5 秒 | 引导关注 |

### 互动引导

1. **结束页必须有引导关注**
   ```markdown
   <!-- .slide: type=end -->

   # 感谢观看

   ## 点赞关注不迷路

   作者 | B站/公众号
   ```

2. **适当使用金句页增加传播性**
   - 在关键结论后插入金句
   - 每 5-8 页一个金句
   - 选择容易传播的观点

3. **封面设计要点**
   - 标题有吸引力
   - 副标题说清价值
   - 视觉冲击力强

### 内容密度

| 视频类型 | 内容密度 | 说明 |
|---------|---------|------|
| 教程类 | 中等 | 步骤清晰，留白适当 |
| 分享类 | 较高 | 信息量大，节奏快 |
| 产品类 | 较低 | 视觉为主，文字精简 |

## 五、转换示例

### 示例 1：概念讲解

**原始内容**：
```
Python 异步编程是一种并发编程模式，可以让程序在等待 I/O 操作时执行其他任务，
从而提高程序的整体效率。相比传统的同步编程，异步编程可以显著提升性能。

异步编程的核心概念包括：
1. 协程（Coroutine）：轻量级线程
2. 事件循环（Event Loop）：调度中心
3. Future/Task：异步操作的结果
```

**转换后**：

**第一页**：
```markdown
<!-- .slide: type=content -->

## Python 异步编程

在等待 I/O 时执行其他任务，显著提升性能

- 协程：轻量级线程
- 事件循环：调度中心
- Future/Task：异步结果
```

### 示例 2：对比分析

**原始内容**：
```
同步编程和异步编程的主要区别：
同步编程：代码顺序执行，等待操作完成后继续。简单直观，但效率低。
异步编程：不等待操作完成，直接执行后续代码。效率高，但代码复杂。
```

**转换后**：
```markdown
<!-- .slide: type=two-column -->

## 同步 vs 异步

### 同步编程

代码顺序执行

- 简单直观
- 效率较低

---

### 异步编程

不等待操作完成

- 效率更高
- 代码复杂
```

### 示例 3：代码演示

**原始内容**：
```
下面是一个简单的异步函数示例：
import asyncio

async def hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

asyncio.run(hello())

这段代码会先打印 Hello，等待 1 秒后打印 World。
```

**转换后**：
```markdown
<!-- .slide: type=code -->

## 异步函数示例

```python
import asyncio

async def hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

asyncio.run(hello())
```

### 先打印 Hello，等待 1 秒后打印 World
```

## 六、检查清单

在转换完成后，使用以下清单验证：

### 内容检查

- [ ] 每页不超过 50 字
- [ ] 列表项 3-5 条
- [ ] 一页一观点
- [ ] 标题简洁有力

### 格式检查

- [ ] 幻灯片类型标记正确
- [ ] 标题层级正确
- [ ] 特殊语法使用正确
- [ ] 元数据格式一致

### 结构检查

- [ ] 有封面和结束页
- [ ] 章节分隔清晰
- [ ] 页面数量合适
- [ ] 类型匹配合理

### B站特定检查

- [ ] 结束页有引导关注
- [ ] 封面有吸引力
- [ ] 适当使用金句页
- [ ] 节奏控制合理

## 七、常见问题

### Q1: 内容超过 50 字怎么办？

**A**: 拆分为多页。保持一页一观点。

### Q2: 列表项超过 5 条怎么办？

**A**: 使用金字塔原理归类分组，或拆分为多页。

### Q3: 不知道选什么类型？

**A**: 默认使用 `content` 类型，适用于大多数场景。

### Q4: 代码太长怎么办？

**A**: 精简代码，只保留核心部分，15 行以内。

### Q5: 网格页可以有 6 个卡片吗？

**A**: 不可以。只支持 2x2（4个）或 3x3（9个）。6 个建议用列表页。

---

## 八、总结

1. **核心原则**：一页一观点、金字塔原则、少即是多
2. **类型选择**：根据内容特征匹配最合适的类型
3. **格式规范**：遵循 Markdown 格式规范
4. **B站优化**：注意节奏控制和互动引导

记住：**好的 PPT 不是展示你知道多少，而是帮助观众理解和记住关键信息。**
